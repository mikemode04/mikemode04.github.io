PROJECT 01 — MACHINE LEARNING

Can Machine Learning Predict Currency Movements?

The Research Question

Foreign exchange markets are notoriously difficult to predict. The famous Meese-Rogoff puzzle (1983) showed that even sophisticated economic models couldn't beat a naive "random walk" forecast. But what if we reframe this as a classification problem instead of a regression problem?

Hypothesis: By predicting only the direction of daily EUR/USD returns (up or down) rather than the exact magnitude, we can achieve accuracy significantly above 50% and build a profitable trading strategy.

Methodology

I tested multiple ML classification algorithms against 5 years of EUR/USD daily data (2019-2024):

  • Features: Lagged returns (1, 5, 20 days), RSI(14), MACD signal, Bollinger Band %B, moving average crossovers
  • Models: Logistic Regression, Random Forest, Gradient Boosting (XGBoost), Neural Network (MLP)
  • Validation: Walk-forward analysis — train on 250 days, test on next 20 days, roll forward
  • Benchmark: Random prediction (50% accuracy)

Implementation

Key Insights

54.7% accuracy is statistically significant — a binomial test confirms this beats random chance (p < 0.05). The neural network outperformed linear models by ~2%, suggesting non-linear patterns exist in the data. However, the edge is thin — transaction costs above 3bps per trade eliminate profitability. The model also degraded during high-volatility periods (March 2020: accuracy dropped to ~51%). This isn't a trading system — it's proof that small edges exist, but capturing them in practice is another challenge entirely.

PROJECT 02 — FACTOR MODELS

Building a Factor Attribution Framework

The Problem

When a portfolio outperforms the market, is it alpha (genuine skill) or just factor exposure (loading up on risky small-caps or value stocks)? Academic factor models like Fama-French decompose returns, but implementing them correctly requires careful attention to data alignment, look-ahead bias, and time-varying factor loadings.

Goal: Build a clean Python framework that takes any portfolio's returns and decomposes them into market, size (SMB), value (HML), and momentum contributions — with rolling betas to capture how exposures change over time.

What I Built

  • Data Pipeline: Automated ingestion from Kenneth French's data library + Yahoo Finance
  • Rolling Regression: 252-day rolling OLS with Newey-West standard errors for robust inference
  • Visualization: Time-series plots of factor loadings to see when betas shift

Implementation

PROJECT 03 — DATA ENGINEERING

Macro Data Pipeline with SQL & FRED API

The Problem

Macro trading strategies require clean, consistent data across dozens of economic indicators. Manually downloading CSVs from FRED is tedious, error-prone, and doesn't handle revisions properly (GDP estimates change!). I needed an automated pipeline that updates daily and tracks historical revisions for point-in-time accuracy.

Solution Architecture

  • Ingestion: Python script pulls data from FRED API on schedule
  • Storage: PostgreSQL database with separate tables for each indicator
  • Views: SQL views combine indicators and generate trading signals
-- Unified macro dashboard with derived signals CREATE VIEW macro_signals AS SELECT date, gdp_growth_yoy, cpi_yoy, unemployment_rate, fed_funds_rate, yield_10y - yield_2y AS yield_spread, -- Yield curve inversion = recession warning CASE WHEN yield_10y - yield_2y < 0 THEN 'INVERTED' ELSE 'NORMAL' END AS yield_curve_signal, -- LEI momentum for cycle timing CASE WHEN lei_3m_change > 0 THEN 'EXPANSION' ELSE 'CONTRACTION' END AS lei_signal FROM economic_indicators ORDER BY date DESC;
What This Enables

Real-time regime detection: One query tells me if the yield curve is inverted and where we are in the business cycle. Point-in-time backtesting: By tracking when GDP/CPI estimates were revised, I can backtest strategies using only information available at the time — avoiding look-ahead bias. 20+ years of history: The full database covers 2000-present for all major US macro indicators.

UPDATED APRIL 10, 2026

What's Happening in Tanker Markets?

Product tankers are experiencing a historic moment. To understand why, you need to understand the key dynamic: it's not about how much oil moves — it's about how far it travels.

Daily loadings are actually down 2.7 million barrels per day versus pre-disruption levels. But West-to-East trade (Atlantic to Asia) has surged from 2 million bpd to 3+ million bpd. A cargo from Houston to Singapore takes 3x longer than one from the Middle East. Same barrels, 3x the ships needed.

Tonne-miles are up 3% YoY even though volumes are down. That's why spot rates are at all-time highs. And if the Middle East situation normalizes? Expect a scramble as charterers race to reposition — another rate spike.

Why Clean Products Outperform Crude

VLCCs are correcting from extreme highs, but clean product tankers (LR2, MR) keep strengthening. The reason: refinery geography doesn't match consumption. You can't just reroute crude — refined products must ship from Asian/Middle East export refineries to European and American demand centers. The trade lanes are structurally longer.
